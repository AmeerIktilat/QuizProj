# Examino — מערכת מבחנים בצד הלקוח

פרויקט אקדמי מלא ב־JavaScript טהור. המערכת מאפשרת למורים ליצור ולנהל מבחנים ולסטודנטים לחפש, לבצע ולקבל תוצאות — ללא שרת, באמצעות `localStorage`.

## הפעלה

מכיוון שהפרויקט משתמש ב־ES Modules, יש להריץ אותו דרך שרת מקומי ולא בפתיחה ישירה של הקובץ:

1. פתחו את התיקייה ב־VS Code והפעילו **Live Server** על `index.html`.
2. לחלופין, מתוך תיקיית הפרויקט הריצו `python -m http.server 5500` וגלשו ל־`http://localhost:5500`.

חשבונות הדגמה שנוצרים בהפעלה הראשונה:

| תפקיד | שם משתמש | סיסמה |
|---|---|---|
| מורה | `teacher` | `1234` |
| סטודנט | `student` | `1234` |

## יכולות המערכת

### מורה

- הרשמה והתחברות כמורה.
- יצירה, עריכה, פרסום ומחיקה של מבחנים.
- הגדרת שם, תיאור, קטגוריה, קוד ומשך זמן.
- הוספה, עריכה ומחיקה של שאלות אמריקאיות.
- חיפוש וסינון מבחנים אישיים.
- צפייה בתוצאות סטודנטים ובסטטיסטיקות.
- ייצוא מבחן ל־JSON וייבוא מבחן מ־JSON.

### סטודנט

- הרשמה והתחברות כסטודנט.
- חיפוש מבחן לפי שם, קטגוריה או קוד.
- ביצוע מבחן עם טיימר, מד התקדמות והגשה אוטומטית.
- קבלת ציון מיידי וסקירת כל התשובות.
- צפייה בהיסטוריה, ממוצע, ציון שיא וכמות מבחנים שעברו בהצלחה.

### תוספות

- מצב כהה הנשמר בין ביקורים.
- עיצוב RTL רספונסיבי למחשב ולטלפון.
- נתוני הדגמה להפעלה ראשונה.
- הרשאות וניתוב לפי תפקיד.
- טיפול בקלט משתמש ומניעת הזרקת HTML בתצוגה.

## עקרונות תכנון

- **OOP:** הישויות המרכזיות מיוצגות במחלקות עם אחריות ברורה. `Teacher` ו־`Student` יורשות מ־`User`.
- **ES Modules:** כל מחלקה ושירות נמצאים במודול עצמאי ומיובאים לפי הצורך.
- **Service Layer:** פעולות אחסון, אימות, מבחנים ותוצאות מופרדות מה־UI.
- **SPA Routing:** המעבר בין הדפים מתבצע בצד הלקוח באמצעות כתובות Hash.
- **JSON:** הנתונים נשמרים ומיובאים/מיוצאים בפורמט JSON.
- **localStorage:** מפתחות המערכת הם `examino_users`, `examino_exams`, `examino_results`, `examino_session` ו־`examino_theme`.
